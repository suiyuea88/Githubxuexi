# GitHub学习馆部署说明

## 推荐方式：Render Web Service

把完整项目上传到 GitHub 后，在 Render 创建 `Web Service`，不要创建成只有静态文件的 `Static Site`。

填写以下内容：

- Language：`Python 3`
- Branch：`main`
- Root Directory：留空
- Build Command：`pip install -r backend/requirements.txt`
- Start Command：`uvicorn backend.main:app --host 0.0.0.0 --port $PORT`
- Health Check Path：`/health`

仓库已经带有 `render.yaml`，也可以选择 Render Blueprint 自动读取部署配置。

## GitHub Token（推荐）

不填写 Token 也能使用，但 GitHub 公共接口的访问次数较少。正式使用时建议在 Render 的 Environment 中添加：

- Key：`GITHUB_TOKEN`
- Value：你在 GitHub 创建的访问令牌

Token 只放在 Render 环境变量中。不要写入 `frontend/config.js`、Python 文件或上传到 GitHub。

## 部署后检查

依次访问：

1. `https://你的域名.onrender.com/health`，应显示 `{"status":"ok"}`；
2. `https://你的域名.onrender.com/projects`，应返回项目数据；
3. `https://你的域名.onrender.com/search?q=Python`，应返回搜索结果；
4. 最后打开网站首页，测试分类、搜索、详情和 README 导学。

## 常见问题

### 页面显示 404

通常是误建成 Static Site，或启动命令填写错误。请确认服务类型为 Web Service，并使用上面的 Start Command。

### 一直显示 Service waking up

Render 免费服务休眠后的首次启动可能较慢。先访问 `/health`，看到 `status: ok` 后再刷新首页。

### 搜索突然失败或没有结果

先检查 `/search?q=Python`。如果提示 GitHub 限流，请配置 `GITHUB_TOKEN`，等待几分钟后重新搜索。

### 首页能打开，但接口是 404

确认 Root Directory 留空，并确认上传的是完整项目，而不是只上传 `frontend` 文件夹。

### 在线体验一直导入

不是所有 GitHub 项目都能在浏览器运行。本站会先检查仓库结构；后端、桌面软件、CLI、Skill 或大型项目应改用 README 导学和本地运行。
